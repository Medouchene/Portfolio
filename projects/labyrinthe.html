Labyrinthe Game

  • Tech Stack: Java and Software design
  • Github URL: Project Link

The objective is to guide a character through a maze, aiming to uncover a hidden treasure while contending with various types of monsters. The project adopts a Scrum-like methodology, emphasizing iterative development through multiple sprints, each initiated with a 2-hour supervised meeting. At the end of each sprint, teams deliver a validated and functional version after deciding on additional features. The project's evolving versions encompass text-based, graphical, labyrinth-incorporated, exit-inclusive, and monster-integrated iterations. Key functionalities include hero movement within the maze, dynamic maze generation from a file or based on selected levels, placement and behavior of randomly generated monsters, and the incorporation of special maze features such as treasure locations, traps, magical effects, and teleportation passages. Combat mechanics are introduced, allowing the hero to be attacked by monsters upon contact and enabling the hero to retaliate by attacking monsters in close proximity or on adjacent cells. The comprehensive list of functionalities encourages students to apply analysis and design principles in the development of an engaging and interactive game.
